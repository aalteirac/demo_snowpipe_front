Let me explain the way this system works. First you need to understand it's a perfect example of useless machine...

Nevertheless, it also demonstrates few interesting mechanisms:

  • Raspberry GPIO control in Docker
  • Raspberry USB ANT+ protocol in Docker
  • Snowpipe Streaming SDK

    • Clicking on the right top button launch a service on the Raspberry and triggers the servo motor (GPIO), it shakes the Octopus on the Lego stand. The lego stand has a speed sensor and send the signal through ANT+ wireless protocol to the same Raspberry (USB ANT dongle). The speed information is sent live to the website and asynchromoulsly streamed to Snowflake with Snowpipe Streaming SDK (Java). The website queries Snowflake to retrieve the speed, display the data on the bottom line chart and calculates the latency.
